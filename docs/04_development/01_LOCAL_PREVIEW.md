1. 说明本地开发和预览的完整流程；

## 3. 开发时预览

### 3.1 启动开发服务器

```bash
npm run dev
```

Vite 默认在 `http://localhost:5173` 启动开发服务器。

### 3.2 特点

- 热模块替换（HMR）：修改代码后浏览器自动更新；
- 快速启动：通常 1–2 秒；
- 源码映射：浏览器 DevTools 中可调试原始 TypeScript 代码。

### 3.3 注意事项

- 开发服务器使用 `@media screen` 样式，不应用 `@media print`；
- 如需预览打印效果，使用浏览器的"打印预览"功能（见第 5 节）；
- 开发服务器的 Tailwind 包含所有工具类（未经过 JIT 裁剪），构建版本更精简。

## 4. 构建后预览

### 4.1 构建

```bash
npm run build
```

### 4.2 启动预览服务器

```bash
npm run preview
```

Vite 默认在 `http://localhost:4173` 启动预览服务器。

### 4.3 特点

- 预览的是生产构建产物（`dist/`）；
- 不包含 HMR，与部署后效果完全一致；
- 包含 Tree Shaking 后的 CSS 和 JS；
- 适用于部署前的最终验证。

### 4.4 适用场景

- 部署前验证构建产物是否正确；
- 检查构建后的文件大小；
- 验证生产环境下的渲染效果。

## 5. 打印预览

### 5.1 浏览器打印预览

在开发服务器或预览服务器打开的页面中：

1. 按 `Ctrl+P`（Windows）或 `Cmd+P`（Mac）打开打印对话框；
2. 选择"另存为 PDF"作为目标打印机；
3. 设置：
   - 纸张尺寸：A4
   - 边距：无
   - 背景图形：开启
   - 页眉和页脚：关闭
4. 查看预览效果。

### 5.2 检查要点

| 检查项 | 说明 |
|---|---|
| 页面尺寸 | 应为 A4（210mm × 297mm） |
| 分页位置 | 封面、章节页是否从新页开始 |
| 表格跨页 | 表头是否在每页重复 |
| 图片完整 | 图片是否被截断 |
| 中文字体 | 字体是否正确显示 |
| 品牌色 | 颜色是否正确 |
| 合规声明 | 声明是否完整、清晰可读 |

## 6. PDF 导出预览

### 6.1 Playwright 导出

```bash
npm run export:pdf
```

导出文件保存在 `exports/` 目录。

### 6.2 检查导出的 PDF

1. 使用系统 PDF 阅读器打开；
2. 逐页检查排版；
3. 检查字体嵌入情况；
4. 检查文件大小（预期 < 10MB）；
5. 检查是否可以正常打印。

## 7. 预览对比矩阵

| 预览方式 | 命令 | 端口 | 适用阶段 | 打印样式 |
|---|---|---|---|---|
| 开发服务器 | `npm run dev` | 5173 | 开发中 | 不应用 |
| 构建预览 | `npm run build` + `npm run preview` | 4173 | 部署前验证 | 不应用 |
| 浏览器打印预览 | 页面中 `Cmd+P` | - | 开发中/验证 | 应用 |
| Playwright PDF | `npm run export:pdf` | - | 最终验证 | 应用 |

## 8. 预览时的常见问题

### 8.1 页面显示空白

- 检查控制台是否有 JavaScript 错误；
- 检查 `index.html` 中的 `<div id="root">` 是否存在；
- 检查 `src/main.tsx` 是否正确挂载。

### 8.2 样式不生效

- 检查 Tailwind 指令是否正确引入（`@tailwind base/components/utilities`）；
- 检查 `tailwind.config.js` 的 `content` 路径是否包含所有源码文件；
- 清除浏览器缓存后重试。

### 8.3 打印预览与屏幕差异大

- 确认 `print.css` 中的规则是否正确；
- 检查 `print-color-adjust: exact` 是否生效；
- 使用 Chrome 浏览器（打印预览兼容性最好）。

### 8.4 字体不正确

- 确认系统是否安装了所需字体；
- 检查 `font-family` 是否正确设置；
- 考虑使用 Web 字体或系统通用字体作为备选。
