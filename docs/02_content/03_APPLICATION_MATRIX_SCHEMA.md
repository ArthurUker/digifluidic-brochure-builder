## 2. 文档目的与适用范围

本文件用于：

1. 定义应用矩阵的数据结构和字段含义；
2. 约束应用矩阵中每行的表达方式；
3. 确保应用矩阵展示符合合规边界；
4. 指导 `src/data/brochure.ts` 中应用矩阵数据的填写。

本文件不替代：

1. 内容策略文档（`00_CONTENT_STRATEGY.md`）；
2. 内容数据模型（`01_CONTENT_MODEL.md`）；
3. 合规文案规范（`04_COMPLIANCE_COPY.md`）。

## 3. 应用矩阵的定位

应用矩阵在白皮书中的作用：

1. 将技术路线与应用场景做可视化对应；
2. 帮助读者快速理解平台的能力覆盖范围；
3. 为不同受众提供针对性的信息索引；
4. 展示平台的广度和技术储备深度。

应用矩阵**不是**：
- 已注册产品的检测项目列表；
- 产品性能参数表；
- 监管审批状态的声明。

## 4. 应用矩阵数据结构

### 4.1 顶层结构

```typescript
export interface ApplicationMatrixData {
  /** 矩阵标题 */
  title: string;
  /** 矩阵说明文字 */
  description?: string;
  /** 矩阵行数据 */
  rows: ApplicationMatrixRow[];
  /** 底部合规声明 */
  disclaimer: string;
}
```

### 4.2 矩阵行 `ApplicationMatrixRow`

```typescript
export interface ApplicationMatrixRow {
  /** 应用方向（如"食品安全"） */
  direction: string;
  /** 产品/材料示例（如"食源性病原体检测芯片"） */
  productExample: string;
  /** 技术路线（如"LAMP / 多重检测"） */
  technologyRoute: string;
  /** 表达重点（面向客户的简要价值说明） */
  keyMessage: string;
  /** 关联的论文引用（可选，填写论文标题关键词或分组名） */
  paperRefs?: string[];
  /** 研发阶段标注 */
  stage: ApplicationStage;
}

/** 研发阶段 */
export type ApplicationStage =
  | 'product'         // 已有产品
  | 'development'     // 研发中
  | 'research'        // 技术储备/研究阶段
  | 'exploratory';    // 探索性方向
```

## 6. 字段填写规范

### 6.1 direction（应用方向）

- 使用简短的中文名称，控制在 6 个字以内；
- 面向客户可理解，如"食品安全"而非"食源性病原体分子检测"。

### 6.2 productExample（产品/材料示例）

- 描述该方向对应的产品形态或材料类型；
- 如尚无对应产品，可使用"XX 检测方案"、"XX 技术储备"等表述；
- 不得虚构不存在或未规划的产品。

### 6.3 technologyRoute（技术路线）

- 列出该方向采用的核心技术方法；
- 多个方法用" / "分隔；
- 保持技术名词准确，如"LAMP"、"RT-qPCR"、"多重 PCR"。

### 6.4 keyMessage（表达重点）

- 面向客户的核心价值说明；
- 控制在 30 字以内；
- 使用合规推荐表达（参见第 7 节）。

### 6.5 stage（研发阶段）

必须如实标注研发阶段：

| 阶段 | 含义 | 适用场景 |
|---|---|---|
| `product` | 已有正式产品 | 已经公司确认可对外介绍的正式产品方向；涉及医疗器械注册的，以正式注册文件为准 |
| `development` | 研发中 | 已有明确产品规划、正在开发 |
| `research` | 技术储备/研究阶段 | 已有论文支撑，但尚未进入产品化 |
| `exploratory` | 探索性方向 | 早期研究，未来可能拓展 |

### 6.6 合规声明

应用矩阵底部必须包含以下声明：

> 上述矩阵为宣传材料结构草案，不代表所有方向均已有正式注册产品。具体产品名称、注册状态和适用范围需以公司正式资料和注册文件为准。

## 7. keyMessage 表达规范

### 7.1 推荐表达

- "支持XX场景的快速检测"
- "展示了平台在XX方向的应用潜力"
- "有助于构建多靶标检测能力"
- "可作为XX场景的技术参考"
- "适配XX检测需求"

### 7.2 禁止表达

- "已完全覆盖XX领域"
- "全球领先的XX检测方案"
- "保证XX检测准确率"
- "唯一能够实现XX的平台"
- "经XX权威机构认证"（除非能提供正式认证文件）

## 8. 应用矩阵的展示方式

### 8.1 Web 预览

- 以表格形式展示，列包括：应用方向、产品/材料示例、技术路线、表达重点；
- 可按研发阶段分组或使用颜色/标签区分阶段；
- 表格应响应式设计，在 A4 宽度内完整展示。

### 8.2 PDF 输出

- 表格应适配 A4 页面宽度；
- 注意分页控制，避免表格行跨页截断；
- 合规声明必须完整保留，且与表格在同一页面或紧接下一页。

## 9. 当前阶段不纳入

1. 动态更新的应用矩阵（当前为静态数据）；
2. 各方向的市场规模数据；
3. 竞品对比矩阵；
4. 各方向的详细技术参数；
5. 交互式筛选或排序功能。

## 10. 风险与控制措施

| 风险 | 控制措施 |
|---|---|
| 研发阶段标注不准确 | 以公司内部产品规划为准，定期核对更新 |
| keyMessage 越界为性能承诺 | 每次新增或修改后按本文件第 7 节检查 |
| 产品示例名称与注册名不一致 | 以公司正式产品命名为准 |
| 合规声明遗漏 | 矩阵底部声明为必填字段，类型系统中 `disclaimer` 为 `string`（非可选） |

## 12. 后续维护规则

1. 新增应用方向时，需同时评估其研发阶段和对应论文支撑；
2. 研发阶段变化时，及时更新 stage 字段；
3. 应用矩阵变更后需重新生成 PDF 并检查表格排版；
4. 每次内容审阅时重点检查 keyMessage 是否合规。
