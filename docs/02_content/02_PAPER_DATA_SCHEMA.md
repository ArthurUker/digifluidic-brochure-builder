## 2. 文档目的与适用范围

本文件用于：

1. 定义论文证据链中每篇论文的数据字段和录入规范；
2. 约束论文数据的展示方式；
3. 确保论文相关表述符合合规边界；
4. 指导 `src/data/brochure.ts` 中论文数据部分的填写。

本文件不替代：

1. 内容策略文档（`00_CONTENT_STRATEGY.md`）；
2. 内容数据模型（`01_CONTENT_MODEL.md`）；
3. 合规文案规范（`04_COMPLIANCE_COPY.md`）。

## 3. 论文数据的定位

论文证据链在白皮书中的作用：

1. 展示 Digifluidic 核心技术和团队的技术积累深度；
2. 为应用场景提供科学研究背景支撑；
3. 说明平台技术路线的学术基础；
4. 为合作方和项目评审方提供可信的技术参考。

论文数据**不是**：
- 产品性能的直接证明；
- 注册审批依据；
- 产品宣传的性能承诺。

## 4. 论文数据结构

### 4.1 顶层结构

```typescript
export interface PaperEvidenceData {
  /** 章节标题 */
  title: string;
  /** 章节总体说明 */
  description?: string;
  /** 按研究方向分组的论文列表 */
  groups: PaperGroup[];
  /** 底部合规声明 */
  disclaimer: string;
}
```

### 4.2 论文分组 `PaperGroup`

```typescript
export interface PaperGroup {
  /** 研究方向名称 */
  groupName: string;
  /** 该方向的研究概况说明（1-2 句） */
  description?: string;
  /** 论文列表 */
  papers: PaperItem[];
}
```

### 4.3 单篇论文 `PaperItem`

```typescript
export interface PaperItem {
  /** 论文标题（完整标题） */
  title: string;
  /** 作者列表（可选，格式："第一作者 et al." 或完整列表） */
  authors?: string;
  /** 发表期刊全称 */
  journal?: string;
  /** 发表年份 */
  year: number;
  /** DOI 编号（可选） */
  doi?: string;
  /** 论文类型 */
  type: PaperType;
  /** 与本产品/平台的关系说明（1-2 句） */
  relevance: string;
}

/** 论文类型 */
export type PaperType =
  | 'journal_article'  // 期刊论文
  | 'review'           // 综述
  | 'conference'       // 会议论文
  | 'patent';          // 专利
```

## 5. 当前论文研究方向分组

根据已有技术积累，当前论文按以下方向分组：

| 分组 | 说明 | 典型论文方向 |
|---|---|---|
| 食源性病原体检测方向 | 食品安全快速检测 | 数字微流控 LAMP、通道-数字混合平台、比色 LAMP |
| 呼吸道病原体检测方向 | 呼吸道感染综合检测 | 多重 RT-qPCR、宏通道到数字微流控、样本到结果系统 |
| 临床评价与专项检测方向 | POCT 和专病检测 | SARS-CoV-2/流感 POCT、HPV 电化学传感、电穿孔系统 |
| 微流控底层技术方向 | 芯片工程和基础研究 | 液滴分裂、卫星液滴控制、单细胞培养、电极寿命优化 |

## 6. 论文数据录入规范

### 6.1 标题

- 使用论文原始标题（中文论文用中文标题，英文论文用英文标题）；
- 不得擅自修改或翻译论文标题。

### 6.2 作者

- 可选项；
- 如填写，建议使用"第一作者 et al."格式；
- 如论文作者中包含公司核心团队成员，可标注"含团队核心成员"。

### 6.3 期刊

- 使用期刊标准全称；
- 英文期刊使用英文名；
- 可标注期刊影响因子分区（如"中科院一区"），但不得夸大。

### 6.4 年份

- 四位数字，如 `2020`；
- 按年份降序排列（最新在前）。

### 6.5 relevance 字段（关键合规字段）

**核心原则**：relevance 字段说明论文与产品/平台的关系，必须保持合规表达。

**正确写法**：
- "展示了平台在食品安全快速检测方向的应用潜力"
- "为多重病原体检测提供了方法学基础"
- "在相关研究中验证了微流控芯片的多靶标检测能力"
- "可作为现场检测方案的技术参考"

**禁止写法**：
- "证明了 VHunter Plus 的检测能力"（将论文等同于产品性能）
- "该论文验证了产品已完全满足临床需求"（夸大、绝对化）
- "权威证明该平台的检测准确率 100%"（绝对化、虚构数据）

### 6.6 合规声明

每组论文数据末尾应包含统一声明：

> 以上文献内容仅作为技术研究基础和应用方向说明，不构成对具体产品性能的承诺。具体产品功能、性能指标、适用范围和注册状态以公司正式资料和注册文件为准。

## 8. 论文数据的展示方式

### 8.1 Web 预览

- 按研究方向分组展示；
- 每组显示研究概况和论文列表；
- 论文以列表形式展示：标题 + 期刊 + 年份 + 关系说明；
- DOI 如有，可添加链接。

### 8.2 PDF 输出

- 与 Web 预览结构一致；
- 注意分页控制，避免单篇论文跨页截断；
- 合规声明必须完整保留。

## 9. 当前阶段不纳入

1. 论文全文 PDF 嵌入或链接；
2. 自动联网获取论文元数据；
3. 论文数据库同步；
4. 论文引用量或影响因子实时数据；
5. 论文全文翻译。

## 10. 风险与控制措施

| 风险 | 控制措施 |
|---|---|
| relevance 表述越界 | 每次新增或修改论文数据后，按本文件第 6.5 节检查 |
| 论文信息过期或不准确 | 定期（建议每半年）核对论文列表，更新发表信息 |
| 论文分组逻辑不清 | 按研究方向分组，每组 3–8 篇，过多考虑拆分 |
| 合规声明遗漏 | 每个 PaperGroup 末尾和整个 paperEvidence 末尾都必须包含声明 |
