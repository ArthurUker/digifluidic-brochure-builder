## 2. 文档目的与适用范围

本文件用于：

1. 集中定义白皮书/宣传册中所有合规声明和免责声明的文案；
2. 统一合规表达口径，避免不同页面使用不同表述；
3. 约束内容编写者的表达边界；
4. 作为 `src/data/brochure.ts` 中合规声明数据的填写依据。

本文件不替代：

1. 公司正式合规文件或法务审核意见；
2. 产品注册文件；
3. 内容策略文档（`00_CONTENT_STRATEGY.md`）；
4. 论文数据规范（`02_PAPER_DATA_SCHEMA.md`）。

## 3. 合规声明体系

白皮书中合规声明分为三个层级：

| 层级 | 位置 | 作用 |
|---|---|---|
| 全局声明 | 白皮书末尾（独立章节） | 整体法律声明，覆盖全部内容 |
| 章节声明 | 论文证据链、应用矩阵等章节末尾 | 针对特定内容的合规提醒 |
| 行内声明 | 产品参数、性能描述附近 | 即时提醒具体数据的适用范围 |

## 4. 全局合规声明

### 4.1 标准版

白皮书末尾必须包含以下合规声明章节：

```text
# 合规声明

本文档为 Digifluidic VHunter Plus 微流控核酸检测平台的技术应用说明材料，旨在向客户、合作伙伴和相关方介绍公司技术平台、产品体系和应用方向。

本文档不作为产品注册文件、产品说明书或产品性能承诺书。

具体产品功能、性能指标、适用范围和注册状态以公司正式产品资料和经审批的注册文件为准。

本文档中的文献引用仅作为技术研究基础和应用方向说明，不构成对具体产品性能的承诺。

未经公司书面许可，不得将本文档内容用于产品注册、广告审批或其他需要正式文件支撑的用途。

如有疑问，请联系珠海市迪奇孚瑞生物科技有限公司。
```

### 4.2 精简版（适用于单页 PDF 末尾）

```text
本文档为技术应用说明材料。具体产品功能、性能指标和注册状态以公司正式资料和注册文件为准。
```

## 5. 章节级合规声明

### 5.1 论文证据链章节声明

用于 `PaperEvidenceData.disclaimer` 字段：

```text
以上文献内容仅作为技术研究基础和应用方向说明，不构成对具体产品性能的承诺。具体产品功能、性能指标、适用范围和注册状态以公司正式资料和注册文件为准。
```

### 5.2 应用矩阵章节声明

用于 `ApplicationMatrixData.disclaimer` 字段：

```text
上述矩阵为宣传材料结构草案，不代表所有方向均已有正式注册产品。具体产品名称、注册状态和适用范围需以公司正式资料和注册文件为准。
```

### 5.3 产品介绍章节声明

用于产品介绍章节末尾：

```text
以上产品介绍基于当前可公开信息。产品功能、性能和注册状态可能因研发进展和监管审批发生变化，最新信息以公司正式发布为准。
```

## 6. 行内合规声明

### 6.1 产品性能参数

当页面展示产品性能参数时，应附加说明：

```text
* 以上参数为典型值，具体性能以产品说明书和注册文件为准。
```

### 6.2 检测应用场景

当描述检测应用场景时，应附加说明：

```text
* 该应用方向基于现有技术储备和研究基础，不代表已获得相关监管批准。
```

### 6.3 临床相关表述

当涉及临床检测、POCT 等表述时，应附加说明：

```text
* 临床应用的注册状态以药品监督管理部门审批为准。
```

## 7. 推荐表达与慎用表达对照

### 7.1 产品能力描述

| 慎用/禁用 | 推荐替代 |
|---|---|
| VHunter Plus 可以检测XX | VHunter Plus 平台可围绕XX检测方向开展方法学适配与应用开发 |
| 产品已完全验证 | 产品已完成内部技术验证 |
| 检测准确率100% | 在实验条件下展示出较高的检测准确性 |
| 全球领先 | 在微流控核酸检测领域具有技术优势 |
| 唯一方案 | 提供了一种创新的检测方案 |
| 金标准 | 参考方法 |
| 全面替代 | 可作为传统方法的补充或替代选项之一 |

### 7.2 论文与技术关系描述

| 慎用/禁用 | 推荐替代 |
|---|---|
| 该论文证明了产品性能 | 该论文展示了平台在XX方向的技术基础 |
| 论文结果等同于产品性能 | 论文结果为产品开发提供了方法学参考 |
| 已发表论文验证了产品 | 已发表论文为相关技术路线提供了研究支撑 |

### 7.3 市场与商业表述

| 慎用/禁用 | 推荐替代 |
|---|---|
| 市场占有率第一 | 在XX细分领域积极拓展市场 |
| 客户遍及全球 | 已与国内外多家机构建立合作 |
| 零投诉 | 持续优化产品和服务质量 |

## 8. 合规声明维护流程

### 8.1 新增合规声明

1. 确认声明内容的合规必要性；
2. 在本文件中新增声明条目；
3. 在 `src/data/brochure.ts` 中引用；
4. 在相关组件中渲染；
5. 经内部合规负责人确认。

### 8.2 修改合规声明

1. 识别需要修改的声明；
2. 在本文件中更新文案；
3. 同步更新 `src/data/brochure.ts`；
4. 重新生成 PDF 并确认声明完整；
5. 记录变更原因。

### 8.3 合规声明检查

每次发布前，需检查：

1. 全局声明是否在白皮书末尾完整呈现；
2. 各章节级声明是否未遗漏；
3. 所有产品性能参数附近是否有行内声明；
4. 声明文案是否与本文件一致；
5. PDF 导出后声明是否可读、未被截断。

## 9. 合规边界红线

以下行为严格禁止：

1. 在未获公司书面批准的情况下，声称产品已获得某监管机构认证或注册；
2. 引用未公开发表或未经授权的论文数据；
3. 将论文中的灵敏度、特异性等数值直接作为产品性能参数展示；
4. 使用"经XX权威机构认证"但无法提供正式认证文件的表述；
5. 在合规声明中使用过小字号、低对比度颜色，使其难以辨认；
6. 在 PDF 导出中裁剪或隐藏合规声明。

## 10. 当前阶段不纳入

1. 法律意见书或法务审核文件（本文件为内部规范，不替代正式法律审核）；
2. 英文版合规声明（当前阶段仅维护中文）；
3. 各国家/地区特定法规要求（当前仅面向中国大陆市场）。

## 11. 风险与控制措施

| 风险 | 控制措施 |
|---|---|
| 合规声明文案过时 | 每次产品注册状态变化时检查并更新 |
| 不同页面声明不一致 | 所有声明从本文件集中引用，组件不自行编写声明文案 |
| PDF 导出后声明丢失 | PDF QA 检查清单中包含合规声明完整性检查 |
| 行内声明被遗漏 | 在内容审阅时逐页检查产品参数附近是否有声明 |

1. 产品注册状态变化时，第一时间更新本文件中的相关声明；
2. 新增宣传材料类型时，评估是否需要新增合规声明模板；
3. 法规或监管要求变化时，全面审查并更新合规声明体系；
4. 每次对外发布前，由合规负责人确认本文件中的声明仍然有效。
