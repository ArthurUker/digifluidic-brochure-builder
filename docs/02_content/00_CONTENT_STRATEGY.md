## 2. 文档目的与适用范围

本文件用于：

1. 定义白皮书/宣传册的整体内容策略和信息架构；
2. 统一内容定位、目标受众和信息传递目标；
3. 指导内容数据文件的设计和维护；
4. 约束内容表达的合规边界；
5. 为内容模型、论文数据结构和应用矩阵设计提供上位依据。

本文件不替代：

1. 内容数据模型（`01_CONTENT_MODEL.md`）；

3. 应用矩阵规范（`03_APPLICATION_MATRIX_SCHEMA.md`）；
4. 合规文案规范（`04_COMPLIANCE_COPY.md`）。

## 3. 内容定位

### 3.1 材料定位

白皮书/宣传册定位为：

> Digifluidic VHunter Plus 微流控核酸检测平台——产品文献应用与技术验证白皮书

核心逻辑：
1. 不仅展示产品参数；
2. 不仅堆砌论文列表；
3. 将科研证据、平台能力、应用场景和产品生态整合为面向市场的可信说明材料。

### 3.2 内容主线

白皮书采用以下主线组织内容：

1. **公司与平台简介**：公司背景、技术平台概览；
2. **VHunter Plus 自动化核酸分析仪**：核心硬件介绍；
3. **微流控芯片与试剂耗材体系**：耗材生态；
4. **技术路线**：LAMP、RT-qPCR、多重 PCR、电化学传感等方法学基础；
5. **应用场景**：食品安全、呼吸道病原体、公共卫生、临床 POCT、专项检测；
6. **论文证据链**：科研文献支撑体系；
7. **平台能力与应用特点**：围绕自动化流程、多靶标检测、现场应用潜力和快速检测需求进行说明，避免写成未经确认的绝对化性能承诺；
8. **应用矩阵**：技术路线 × 应用场景对应关系；
9. **合规说明**：产品注册状态、适用范围声明；
10. **联系方式与后续合作入口**。

## 4. 目标受众与内容侧重

| 受众类型 | 内容侧重 |
|---|---|
| 医疗与临床相关方 | 自动化流程、样本到结果、检测通量、应用场景 |
| 食品安全与监管方 | 食源性病原体检测、现场快速筛查、平台适配性 |
| 公共卫生与应急方 | 多病原体综合检测、自动化、便携化、快速响应 |
| 科研与产业合作方 | 微流控平台能力、方法学扩展、芯片开发潜力 |
| 政府/项目申报方 | 技术基础、论文证据链、产品化路径、应用价值 |
| 内部评审人员 | 宣传口径、内容准确性、页面质量、发布风险 |

## 5. 内容组织原则

### 5.1 模块化

- 每个章节独立成段，可单独阅读；
- 章节间通过"上一章/下一章"或锚点导航连接；
- 单章内容控制在 1–3 页 A4 以内。

### 5.2 层次化

- 每个章节采用"概述 → 详细说明 → 关键要点"结构；
- 技术内容先给出简明结论，再展开技术细节；
- 重要数据和关键表述加粗或使用信息框突出。

### 5.3 可复用

- 同一应用场景的描述在不同材料中保持一致；
- 合规声明统一维护，各页面引用同一来源；
- 技术参数以公司正式资料为准，不在多处重复定义。

### 5.4 可维护

- 所有内容集中在 `src/data/brochure.ts`；
- 修改一处即可影响所有引用该数据的组件；
- 新增章节只需在数据文件中追加条目、在组件树中添加对应组件。

## 6. 内容维护方式

### 6.1 主数据源

当前阶段使用 TypeScript 数据文件：

```text
src/data/brochure.ts
```

该文件包含：
- 封面信息；
- 各章节标题和正文；
- 论文证据列表；
- 应用矩阵数据；
- 合规声明文案；
- 联系方式。

### 6.2 维护流程

1. 编辑 `src/data/brochure.ts`；
2. 本地 `npm run dev` 预览效果；
3. 检查 Web 预览和打印预览；
4. 确认无误后提交。

### 6.3 未来扩展方向

- 多语言支持：可为每种语言创建独立数据文件，如 `brochure.en.ts`；
- JSON 化：如需非开发人员维护内容，可将数据迁移为 JSON 文件；
- 当前阶段不引入 CMS。

## 7. 合规表达边界

### 7.1 核心原则

1. 区分科研论文、技术储备、产品能力和注册声明；
2. 不得将论文结果直接写成产品性能承诺；
3. 不得使用夸大宣传和绝对化表达；
4. 具体产品功能、性能指标和注册状态以公司正式资料为准。

### 7.2 推荐与慎用表达

| 慎用/禁用 | 推荐替代 |
|---|---|
| 权威证明 | 研究显示、文献报道 |
| 完全验证 | 在相关研究中展示了 |
| 绝对领先 | 具有某方面优势 |
| 保证检测 | 可用于支持 |
| 唯一方案 | 提供了一种方案 |
| 全球第一 | 在相关领域具有创新性 |
| 零风险 | 在规范操作下 |
| 百分百准确 | 具有较高的准确性 |

## 8. 当前阶段不纳入内容

1. 产品注册证书扫描件（需公司正式审批后决定）；
2. 临床评价详细数据（需合规确认）；
3. 价格和商务条款；
4. 竞品对比（当前阶段不纳入）；
5. 用户案例和推荐语（需客户授权）；
6. 实时更新的产品参数（以静态内容方式维护）。

## 9. 风险与控制措施

| 风险 | 控制措施 |
|---|---|
| 论文表述越界为产品承诺 | 严格使用推荐表达方式，审阅时重点检查 |
| 内容与实际产品不符 | 以公司正式产品资料为唯一依据 |
| 版本混乱 | 数据文件纳入版本控制，每次发布标注版本号 |
| 多语言内容不一致 | 当前仅维护中文版，未来多语言时建立同步机制 |
